왜 Ground Truth가 필요한가
🔍
정량적 평가의 구조적 필요성
시나리오 설명만으로는 LLM의 탐지 성능을 수치로 측정할 수 없습니다. GT는 평가 기준이자 정량화의 근거입니다.
⚙️
자동 GT 생성
PoC 기반 공격 시나리오 실행 중 발생하는 Windows Event Log / Sysmon 이벤트를 단계별로 수집해 JSON 형식의 GT를 자동 생성합니다.
📊
3가지 평가 지표
Stage-level Accuracy, Sequence Similarity(LCS 기반), Scenario-level Accuracy를 통해 다각도 정량 평가를 수행합니다.
"시나리오는 설명이고, GT는 평가 기준이다. GT 없이는 정량적 평가가 구조적으로 불가능하다."
GT 데이터 형식
{
  "stage": "초기 접근",
  "expected_events": [4624, 4648, 4625],
  "log_source": "Windows Event Log"
}

자주 묻는 질문
GT(Ground Truth) 없이는 LLM의 디지털 포렌식 로그 분석 성능을 정량적으로 측정할 수 없습니다. 본 프레임워크는 GT 자동 생성을 통해 이 구조적 한계를 해결합니다.
시나리오 유형에 따라 Windows Event Log와 Sysmon 로그를 선택적으로 사용합니다. 로그 소스 선택은 Sysmon에 한정되지 않고 시나리오 특성에 따라 결정됩니다.
PoC 공격 시나리오 실행 중 발생하는 이벤트를 단계별로 캡처해 JSON 형식으로 저장합니다. 이벤트 ID 기준이 아닌 행위 수준(behavior-level) 매칭을 평가 기준으로 사용합니다.
GT 이벤트 시퀀스와 LLM이 탐지한 시퀀스 간 LCS(최장 공통 부분수열)를 계산해 유사도를 산출합니다. 이벤트 순서 보존 여부까지 반영하는 지표입니다.
NIST CFReDS 데이터 유출 시나리오를 기반으로 구성했습니다. VM 환경에서 6단계 공격을 직접 실행해 실제 로그와 GT를 생성했습니다.
